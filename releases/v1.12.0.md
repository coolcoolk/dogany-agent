# Release notes -- v1.12.0 (2026-07-18)

Type: MINOR (new install-time agent-class split surface + 6 delivery slices;
no breaking changes to existing main-class instances).

## Summary

Install-time main/domain agent-class split (DGN-227). Agents now mint as
"main" (lifekit-bearing, aggregating) or "domain" (specialist, standalone).
Six implementation slices landed as a single merged branch
(`feat/dgn227-install-split`, 13 commits, harness 243/243 green,
merged to main at 39274f4).

DGN-419 Part B (actual Warg pack publish -- live data boundary, requires
separate owner gate) is NOT in this release. The publish pipeline infrastructure
(Part A) ships; the first official pack entry does not. Do not claim a pack
shipped in release notes or user-facing communications for this version.

## Slices

### DGN-416: install.sh hardening

MAJOR-1: fresh domain agents now seed `LIFEKIT=off` unconditionally over any
`pending` scaffold value (`conf_set_if_absent` promoted to `conf_upsert` for
the domain-off seed path). Prevents lifekit setup prompt on first domain
session.

MAJOR-2: main-add class guard added. `flow_main_add_finalize` now calls
`registry_count_class main` before writing the registry entry, enforcing the
single-main invariant. Without this, a domain user selecting "add main" could
silently overwrite the domain registry row.

MAJOR-3: crash-and-resume for domain installs. `step_agent_class` now loops on
`AGENT.md`-exists occupancy (interactive: re-prompt slug; same-canonical-root:
pass through to re-mint; foreign root: hard-exit unchanged). Fixes the
dead-end where a mid-install crash left the user unable to resume without
knowing `--root`.

MINOR-3: blank-catalog fallback now prompts for confirmation rather than
silently auto-selecting.

MINOR-7: `DOGANY_GATE_LOADED_OVERRIDE` and `DOGANY_LAUNCHD_CAPTURE` production
shims gated behind `DOGANY_INSTALL_LIB=1` (test-only flag). Prevents accidental
gate bypass in production installs.

MINOR-8: `_stamp_role` called as a subcommand now returns non-zero on missing
`AGENT.md`, keeping the install fail-closed on partial mint state.

Intermediate grill (Opus, after DGN-416+417 combined): caught FATAL-1
(domain LIFEKIT seed no-op due to pending scaffold), fixed in commit 1a6b134.
Final harness: 72/72 green.

### DGN-417: mint/update propagation

MAJOR-4 (mint.sh): `DOGANY_AGENT_CLASS` and `DOGANY_PACKS` added to the
keep-if-present block. Re-minting a domain instance no longer silently resets
its class to the main default.

MAJOR-5 (update.sh): `plists.defer` estate substitution. The routine-rename
loop in `update.sh` section 3b now applies the same `telegram-agent` ->
`<agent-name>` substitution to `plists.defer` entries that it already applied
to `.plist` files. Without this, a framework update that delivered
generic-brief plists could activate briefings on all live instances via pack
install STEP 10.

Propagation note: `install.sh`, `scripts/mint.sh`, `update.sh` are
canonical-only installer scripts -- they are NOT vendored to instances.
Template `routines/` and `skills/` reach live instances (Metal, Ag, Warg, etc.)
via release consumption (Metal on release channel, Ag on main channel).
No bridge or OSS impact. No live-instance files were hand-edited by this slice.

### DGN-418: pack lifecycle completion

B5: frozen knowledge snapshot delivery channel. `pack_install.sh` STEP 6 now
installs a bundled snapshot from the pack payload (publisher-local snapshot),
enabling knowledge delivery without a live warehouse connection.

B6: net-new skill directory install mode. STEP 7 gains a `net-new` branch that
installs skill directories absent from the instance, with pack-owned preserve
tag registration (D1 compliance).

NM3: `checksums.sha` verification at install time. Mismatch is a loud FAIL;
absent checksum file (legacy/pre-NM3 packs) warns and continues.

MINOR-5: `plists.defer` defer-merge. When a pack ships its own `plists.defer`,
the entries are merge-appended (deduped, section-marked) rather than
overwriting the instance-level manifest. Format-compatible with the DGN-417
MAJOR-5 substitution pass.

### DGN-419 Part A: publish pipeline

`scripts/pack/pack_publish.sh` (new). Three mandatory refinement gates:
1. Personal-data strip: transcripts, memories, `.env`, user-config keys -- loud
   FAIL on any hit.
2. Persona-field blank: `AGENT.md` identity fields zeroed for onboarding
   population at install time.
3. Knowledge-pin gate: verifies a release-pinned knowledge snapshot is present.

`.source-sync` metadata (section sha256, source path omitted -- no personal
data in canonical), `pack/CHANGELOG.md` convention established.

Publish-side `checksums.sha` generation (closes DGN-418 OQ3 round-trip).

Catalog `en` locale fields added; `status != official` packs filtered from
menus.

Harness: 133/133 green on synthetic fixtures (live Warg workspace not
touched -- data boundary maintained).

### DGN-420: briefing runtime

`generic-brief.sh` (new) composes domain agent morning brief, daily retro,
and weekly retro. Submit-mode writes the section to the submit queue
(sender-encoded filename); standalone mode sends directly.

P20 fallback: if submit fails or no main agent picks up within window, the
domain agent falls back to a standalone push with a one-per-day warning push.

MINOR-6 peer discrimination: `handoff_aggregate` per-peer glob now includes
the sender token, preventing N-peer conflation where the first matching file
was attributed to all peers.

i18n: `ko`/`en` locale support via `AGENT_LANG` + `config/i18n/<lang>.json`.

Config-driven fire times: morning 07:00, retro 22:00, weekly Sunday 20:00
(defaults; set by onboarding step in DGN-422).

Role prose defaults (ko/en) for the "life assistant" main-class role seeded
in `generic-brief.sh`.

### DGN-421: mode transition

`brief-slot-ctl.sh` (new): manages launchd slot scheduling without touching
the brief composition files (pure scheduling layer).

G2-5 standalone-to-submit transition gate: verifies the domain section is
written and accepted before flipping `BRIEF_ROUTING` to submit mode. Reverse
transition (main removed) flips back.

E2-2 timing: domain brief generation is scheduled to lead main aggregation by
`GEN_LEAD_MIN` (default 45 min, following the Ag/Warg live precedent);
configurable via `DOGANY_GEN_LEAD_MIN`.

CS-1 (final grill catch, commit d85246f): `adjust_domain_slot_time` now
updates `BRIEF_TIME_<slot>` in `config/agent.conf` alongside the launchd plist,
eliminating wording-clock / fire-clock drift.

### DGN-422: onboarding + v2 path alignment

Q6 role-question conditional retention aligned across three template copies:
`agents/.template/AGENT.md`, `skills/dogany-user-onboarding/SKILL.md`,
`routines/onboarding-check.py`. Judgment key: `Primary-focus` placeholder
presence.

Regression battery (R17): fresh mint / migration / legacy preserve-tag paths
all exercise the skills-category refine branch, STEP 3 migration branch, and
`lib/prompts` accounting point under real `pack_install.sh` invocation (not
stubs).

G3-b mint-agent path: `--peer-main` flag added to `scripts/mint.sh`;
`HANDOFF_PEER_MAIN` (briefing axis) separated from migration peer detection
axis (`MIGRATION_PEER`), preventing onboarding migration-detection false
positives when a main agent is declared post-domain-mint.

Briefing-time onboarding step (420 seam): `set-briefing-times.sh` (new) writes
`BRIEF_TIME_*` keys to `config/agent.conf` and regenerates the plist
`StartCalendarInterval` values. Called at domain-agent first contact.

Bug fix: `_conf_upsert` used a single-key `grep -v` under `set -e` that could
silently kill the script on a non-matching key; fixed.
