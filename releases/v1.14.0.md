## Fixed

### DGN-503: diet-log English step-narration leak guard

`agents/.template/.claude/skills-bundle/diet-log/SKILL.md` gains an explicit
guard line suppressing English internal-planning narration in the
portion-scaling and composite-record flow.

Root cause: the model was leaking internal English planning lines (e.g. step
annotations, intermediate scaling notes) into user-facing replies during the
`--scale` and per-row `--new` insert flows. The guard makes scaling and insert
steps silent internal mechanics; only the final record summary in the
configured user language is presented to the user.

Instances consume via `update.sh`. No data-model change; no migration.

## Changed

### DGN-360: template cron queue class set to heavy for consolidate/classify/mirror-reconcile

Template LaunchAgent plists and the Linux mirror-reconcile `.service` unit
updated to set `queue_class=heavy` for the three Claude-heavy cron jobs:

- `consolidate` (04:30 macOS plist)
- `classify-inbox` (05:00 macOS plist)
- `mirror-reconcile` (macOS plist + Linux `.service`)

These jobs compete for the same Claude process slot. Serializing them via the
machine-global cron-guard queue (slots=2, fail-open) prevents stampede-style
failures on fresh mints. Live instances were assigned the class directly on
2026-07-22 (estate-wide); this change makes it the template default so new
mints are correct from birth.

---

### DGN-494: extract_options keeps buttons when reply carries multiple numbered lists

`agents/.template/bridge/options.py` updated. Previously `extract_options`
took the first contiguous `1..N` numbered run found in a reply, so a prose
"here are your choices:" lead-in list before the `[[OPTIONS]]` block would
satisfy the extractor early and the actual option buttons were silently
discarded.

Fix: take the last contiguous `1..N` run. The stray-1-then-gap guard
(DGN-085 / DGN-325) is preserved: a length-1 run that is not part of a
contiguous sequence is still discarded. Test `test_dgn494_multi_list` (130
lines) added; all existing frozen assertions remain green.
