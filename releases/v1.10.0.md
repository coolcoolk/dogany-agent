# Release notes -- v1.10.0 (2026-07-18)

Type: MINOR (DGN-387 adds new artifacts and a new update channel; DGN-390
adds config-driven budget control; DGN-391/395 are fixes).

## Summary

AGENT.md template diet campaign (DGN-387 + DGN-390): new cold ops reference
doc (`AGENT-OPS.md`), new `update.sh` 3k2 refresh channel, procedural
boilerplate removed from the hot-injected `AGENT.md`, and session-recap
injection budget made config-driven. Combined recurring saving: ~550-620
tok/turn per future mint; ~1,650-1,850 tok/turn net across the live estate
after migration. Plus skill-creator RULES conflict fix (DGN-391) and
cron-guard/push.sh reliability fixes (DGN-395).

## Added

### DGN-387: AGENT.md template diet -- AGENT-OPS.md cold doc + update.sh 3k2 channel

Background: every minted instance's `AGENT.md` is hot-injected every turn.
~50-80 lines (600-1,000 tok/turn per instance) were framework procedural
boilerplate with zero conversation value: self-restart and self-update
procedures, subagent dispatch routing table, framework code boundary clause,
language register guard, CRAFT activation paragraph, and workspace Paths.
Three instances (Warg, Smith, Kojeni) additionally carried raw placeholder
junk (`<__AGENT_EMOJI__>`, `<__LAUNCHD_LABEL__>`) from unsubstituted template
blocks.

Design survived two independent adversarial grills (R1: 2 FATAL-class
findings fixed; R2: 0 FATAL, 6 MAJOR, 6 MINOR -- all folded). G5 final
real-diff grill: PASS-WITH-FIXES (0 FATAL, 1 MAJOR = ERRATUM below,
5 MINOR accepted). Owner approval: dec-057 (2026-07-17 23:37).

What ships in this release (Step 0 of the migration plan):

- New `agents/.template/AGENT-OPS.md` (~45 lines, cold -- never
  hot-injected). Four sections: (1) self-restart full procedure with flagless
  base command; (2) self-update semantics (framework consumption, never a
  release); (3) subagent dispatch routing table including release approval-gate
  sentence; (4) upstream reporting lane pointer. Placeholder contract:
  `__PROJECT_ROOT__` only -- identity-gated tokens forbidden; contract is
  mechanically asserted by the 3k2 update channel.

- Template `AGENT.md` diet: blocks 4/6/7/8/9/10 removed (CRAFT activation,
  framework code boundary, dispatch pointers, self-restart procedure,
  self-update procedure, Paths), replaced with a 3-bullet `### Ops` pointer
  block carrying both emergency breadcrumbs (restart command, self-update
  command) and the release approval-gate line. 217 -> 172 lines on the
  template; equivalent savings vary per live instance (see Token accounting
  below).

- `rules/RULES.md` +2 lines:
  - `## Coding`: framework code boundary with canonical-maintainer and
    sandboxed-PoC exemptions (avoids constitutionally banning Smith's charter
    from its own sandbox).
  - `## Output`: language register guard -- English framework/workflow text
    is internal working material, never a speaking register.

- `scripts/update.sh` 3k2 channel: new section immediately after 3k (RULES
  refresh), handling `AGENT-OPS.md` with post-substitution SHA comparison
  (the skills-channel pattern -- verbatim compare would fire spurious
  user-modified WARN on every mint since the file carries `__PROJECT_ROOT__`).
  Algorithm: template-side contract assert (identity-gated token detect ->
  WARN); dest-adjacent `mktemp` + `cp -pL` + `subst_one`; SHA compare against
  manifest-recorded and on-disk; user-modified backup + atomic `mv -f` refresh.
  IDENTITY GUARD comment reworded (two framework-owned root files now: RULES.md
  and AGENT-OPS.md). Root-md sanity grep extended with generic
  `__[A-Z][A-Z_]*__` pattern (catches the `__AGENT_EMOJI__` / `__LAUNCHD_LABEL__`
  class that the previous named alternation missed).

- `scripts/mint.sh`: mandatory framework-manifest recording at mint time
  (AGENT-OPS.md + RULES.md SHA written to `.claude/.dogany-framework.sha`).
  Without this, every fresh mint would land on the no-entry branch of 3k2,
  causing a spurious user-modified WARN + stray backup on the first
  self-update even with a pristine instance. CRAFT activation note relocated
  to `mint.sh` Next-steps heredoc.

- Template `.claude/agents/baseline-editor.md`: stamp-lint rule (provenance
  stamps WARNING severity, default action move-to-worklog; instances whose
  stamps are declared authority chains are exempt) + English-ASCII writing
  mandate for workflow and procedure prose.

- `tests/agentops/` acceptance suite (T1-T5): T1 fresh mint + same-version
  update (no WARN), T2 double update quiet, T3 hand-edit WARN + backup,
  T4 IDENTITY_OK=0 byte-identical, T5 cross-version proof (mint at state A,
  update from state B -- no spurious WARN, manifest updated). T1/T2/T5 also
  assert no `.new.*` litter and correct file mode.

ERRATUM (G5-1): do NOT expect an `agents/main/` scaffold diet in this
release. `agents/main/` is gitignored (never tracked in the canonical repo);
the scaffold was already locally dieted but cannot ship via git. Out of scope.

### DGN-390: session-recap injection budget config-driven

`routines/session-recap.py` previously had hardcoded constants `PAIRS=4` and
`CHAR_CAP=1000` (worst-case ~8,000 chars injected at session start,
approaching the full constitution size of ~8,700 chars). The AGENT.md diet
(DGN-387) would have been partially re-expanded by the recap budget --
same campaign, separate surface.

Fix: `RECAP_PAIRS` and `RECAP_CHAR_CAP` are now read from `config/agent.conf`
with silent fallback to defaults of 2 pairs / 500 chars. Worst-case
session-start injection reduced to ~2,000 chars. Template `agent.conf` gains
8 lines of commented keys for instance-level tuning. Existing instances
receive new defaults on the next framework update via the `routines/` rsync
channel; `agent.conf` keys are write-if-absent (existing values never
overwritten).

## Fixed

### DGN-391: skill-creator step 6 RULES conflict -- replace MEMORY.md hand-write

`dogany-skill-creator/SKILL.md` step 6 instructed agents to log a new skill
fact directly to `MEMORY.md`, conflicting with two RULES clauses:
`memories/` is engine-owned, and agents must never hand-write to it. On
2026-07-17 alone, three instances (Kojeni, digear, digear-sh) each
independently applied their RULES reasoning to silent-skip step 6 -- the
skill itself was generating the conflict class it was meant to avoid.

Fix (Option A): new step 6 instructs agents to leave a one-line session note;
the nightly consolidate routine harvests it into long-term memory through the
engine's normal path. No additional tooling required; engine-compatible.
Three copies updated to byte-identical wording (canonical `skills/`, template
via hardlink, Metal live copy). Other live instances ride the next framework
update.

### DGN-395: cron-guard stderr tail + push.sh empty-output retry

Two reliability defects surfaced on 2026-07-17 when a transient auth/limit
window caused `claude -p` to return empty during `ag.daily-retro`.

Defect 1 -- `routines/cron-guard.sh` failure path tailed only the stdout
log, which on real failures contained stale `[push] sent OK` lines (zero
diagnostic value). Fix: the failure path now derives the sibling stderr log
path (`<name>.stderr.log`, launchd `StandardErrorPath` convention) and
prefers it when non-empty; stdout fallback when stderr is absent or empty.
Safe fallback: if the log path does not carry the `.stdout.log` suffix the
derivation is skipped.

Defect 2 -- `routines/push.sh` generation call exited 1 on the first empty
`claude -p` response. Fix: retry 3 attempts total with 5s backoff; on final
failure emit a one-line reason to stderr ("claude returned empty after N
attempts") and exit 1. Survives a single transient empty generation; only
fails after retries exhausted.

Tests: 24 cron-guard tests + 6 push.sh retry tests green on canonical and
all propagated copies. 4-way propagation complete (Metal + Ag + template;
OSS skipped -- routines not in bridge-only repo per propagation ruleset).

## Remote instance instructions (kkari + shawn downstream maintainers)

Migration lag is harmless: old fat `AGENT.md` + new `AGENT-OPS.md` coexist
as redundancy, never a gap. When you are ready:

1. Run `routines/self-update.sh` first (consumes this release).
2. Verify: `AGENT-OPS.md` exists at your instance root and contains no
   dunder tokens (no `__SOMETHING__`); verify `RULES.md` is at the canonical
   version (47 lines with the two new Coding/Output lines).
3. KEEP the `AGENT.md` dispatch-pointers block until your bridge version is
   confirmed to inject subagent descriptions in live transcripts (A1 gate --
   unverified on remote bridges as of this release). If you keep the block,
   it is redundant with `AGENT-OPS.md` but causes no harm.
4. Hand-apply the `baseline-editor.md` stamp-lint and writing-mandate delta
   to your local `.claude/agents/baseline-editor.md`. Agent defs ship at
   mint time only -- `update.sh` never refreshes them.
5. When ready to slim the hot context: back up `AGENT.md`, then delete the
   fat framework blocks (self-restart, self-update, dispatch pointers,
   framework code boundary, language register guard, Paths) and install the
   3-bullet `### Ops` pointer block per the template. Old fat `AGENT.md` and
   new `AGENT-OPS.md` are safe to coexist if you defer.
