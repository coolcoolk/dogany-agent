## Slices

### DGN-441: pack_publish finalize mode + CHANGELOG prepend refactor

New `--mode finalize` for `scripts/pack/pack_publish.sh`. Addresses the
gap where re-sealing a hand-curated pack payload (e.g. dev pack, where
payload diverges intentionally from the live agent upstream) required five
manual steps: refresh-source-sync + CHANGELOG append + checksums regeneration
+ catalog upsert + AGENT.md.add edit. This mode automates those five steps
while leaving the payload content byte-for-byte unchanged.

Key invariants:
- I1 (payload unchanged): finalize writes only seal artifacts (CHANGELOG,
  checksums.sha, catalog). The `<pack-id>/<refslug>/` curated content is
  never touched.
- I2 (gates detect-only): the three B4 gates (personal-data strip, persona
  blank, knowledge-pin) detect and fail; they never mutate the payload.
- I3 (self-exclusion): `checksums.sha` regeneration excludes both
  `checksums.sha` itself and `.source-sync`.

Mode contract: `--mode {snapshot|finalize}` (default: snapshot, backward
compatible). Finalize-exclusive args: `--mode finalize --source-root
<live-root> --pack-id <id> --pack-version <semver> --packs-dir <path>`.
Materialization args (`--section`, `--skill`, etc.) are forbidden in
finalize mode (G-F2 FAIL). Source-sync baseline is read non-destructively
for drift reporting; finalize never regenerates `.source-sync` (F3 fix).

CHANGELOG prepend refactor (DGN-441 F4, applies to both modes): the prior
`cat >` overwrite is replaced by newest-first prepend. Existing entries are
preserved; the new entry is inserted at the top. Fixes a latent
history-destruction bug affecting both snapshot re-seals and finalize.
Harness R21 re-greened.

Design: two grill rounds (Opus); backbone survived both rounds; locked as v2
spec before build. First live use: dev pack 1.1.0 finalize (DGN-449, same
push). 3 commits: 48c891a (finalize + prepend), 7f474c4 (R20/R21 harness),
9e5fa19 (W-1 cleanup).

### DGN-445: harness green recovery (R3 catalog-dynamic version + R5 checksums sync)

merge preparation. Not caused by DGN-441 -- origin/main was already red after

  NM3 working as designed, not a product bug. Fixed by regenerating
  `checksums.sha` from the actual fixture state.

Combined result after merge: 280/280 green (DGN-445 +47 + DGN-441 R20/R21
+37 over the pre-fix baseline of 233 pass). Commit: d4db3aa.

### DGN-449 / dec-073: dev pack 1.1.0 (separate semver axis)

Dev pack version axis is independent of the framework release. Version
1.1.0 (43f2edd) is the first live use of the finalize mode.

Fragment additions in `packs/dev/refdev/AGENT.md.add` (DEV-PACK-FRAGMENT):
- Work-item ticket discipline: every incoming task, backlog entry, and
  pending decision is tracked as a worklog ticket (open > wip > blocked >
  done / parked). No work-tracking to the memory store; memory is for
  durable fact recall only.
- Managed-target registry principle: keep one slug-keyed identity map of
  managed targets (slug / display-name / path or endpoint). Resolve any
  reference to a slug via this map. Single-source separation: identity
  (registry) vs version/skew vs topology are separate concerns; slug is
  the cross-reference key.

to generic form (no estate-specific proper nouns, no personal data).

instances without it (kkari, shawn both require fresh install). The two
remote instances (kkari, shawn) will consume this via the v1.13.0 update
