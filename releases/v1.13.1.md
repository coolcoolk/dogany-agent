## Changed

### DGN-458: skill-creator -- driver-mode hook pattern note upstreamed

`skills/dogany-skill-creator/SKILL.md` adds a "driver-mode hook pattern"
section (34 lines) between the trigger-tiers and tone sections.

Background: the pattern was originally captured in Metal's local skill-creator
(DGN-428/433 origin) and was never upstreamed to the canonical. The omission
went unnoticed until v1.13.0 consumed the canonical via `update.sh` and
overwrote the local copy, causing silent loss. A backup preserved the content;
this patch restores it to the canonical.

Content: guidance for skills that drive a CLI subagent in a persistent loop
(e.g., a REPL or session manager) and need a single-writer session registry.
The pattern uses Claude Code hooks to maintain registry integrity and
documents:
- Why a hook, not in-skill writes: the CLI subagent runs outside the agent
  turn boundary; only a hook fires at the correct lifecycle point.
- How: `stop` hook script writes a session record on every turn end; the skill
  reads the registry and never writes directly.
- What: hook payload fields (session_id, transcript_path, cwd, hook_event_name)
  with an explicit CAVEAT that these are implementation details, not a stable
  API -- avoid building logic that would break on payload changes.

No new machinery shipped. This is a documentation-only addition; existing
skill-creator behavior is unchanged. Instances receive this on the next
`update.sh` run against this release.

---

### DGN-456: baseline-editor.md -- unified canonical (instance-neutral)

`agents/.template/baseline-editor.md` is replaced with a unified definition
that is the union of four previously split rule sets:

| Rule set | Was in |
|:--|:--|
| Writing mandate (English/ASCII only in all workflow output) | .template only |
| Stamp-lint (DGN/dec references belong in worklog, not in AGENT.md text) | .template only |
| Hot-inject discipline (Why-How-What structure; hot inject carries What only) | Metal live only |
| Pack-mirror gate (AGENT.md fragments must mirror the pack source section) | Metal live only |

The four sets are merged verbatim into a single document. All persona tokens
are made generic ("the user's form of address", "the agent's name") -- no
instance-specific proper nouns remain. STAMP-LINT is self-applied: any
DGN/dec stamp in the pack-mirror gate section is condensed to a one-line
rationale pointing to this ticket.

Effect: new agents minted from the template now inherit the complete four-rule
baseline-editor definition. Live instances receive the unified form on the next
`update.sh` run (agent definitions ship at mint only and are not refreshed by
`update.sh`; live instances should apply the unified form manually or at next
re-mint). No behavior change for agents that already had the full rule set.
